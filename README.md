Описание проекта
Название: Агро Платформа
Тип: Единая корпоративная платформа управления агрохолдингом
Суть проекта
Агро Платформа — это современная внутренняя корпоративная система, предназначенная для комплексного управления всеми направлениями деятельности агрохолдинга.
Проект призван заменить разрозненные инструменты, Excel-файлы и устаревшие программы на единую, удобную и масштабируемую платформу. На первом этапе реализуется модуль «Управление земельным банком», который в дальнейшем станет фундаментом для остальных направлений (договоры аренды, мониторинг техники, финансы, отчётность и др.).
Архитектура
Проект построен по архитектуре Modular Monolith (модульный монолит).
Это «золотая середина» между классическим монолитом и микросервисами:

Всё приложение разрабатывается и деплоится как единый процесс
Код строго разделён на независимые бизнес-модули (bounded contexts) с чёткими границами
Высокая внутренняя связность и низкая связность между модулями
Легко поддерживать, тестировать и в будущем выносить отдельные модули в микросервисы

Такая архитектура идеально подходит для постепенного роста проекта и командной разработки.
Технологический стек
Backend:

Python 3.12 + FastAPI
SQLModel + SQLAlchemy 2.0
Alembic (миграции БД)

База данных:

Microsoft SQL Server 2022 (Developer / Standard)

Архитектурные подходы:

Domain-Driven Design (DDD)
Vertical Slice Architecture
Clean Architecture внутри модулей

Инструменты разработки:

uv (управление зависимостями)
Ruff + Pyright (линтинг и типизация)
Docker + docker-compose
Git

Планируется в ближайшее время:

Авторизация и RBAC/ABAC (fastapi-users)
Аудит изменений
Celery + Redis (фоновые задачи)
Интеграция с ESRI ArcGIS

Frontend (в разработке):

Next.js 15 (App Router) + React + ArcGIS JS API
